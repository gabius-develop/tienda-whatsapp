  git clone https://github.com/TU_USUARIO/tiendawhatsapp.git

  npm install

  ---
  PARTE 2 — Configurar Supabase (base de datos)

  2.1 Crear proyecto en Supabase

  1. Ve a https://supabase.com → New project
  2. Elige nombre, contraseña de DB y región (ej: South America - São Paulo)
  3. Espera ~2 minutos a que el proyecto esté listo

  2.2 Ejecutar el SQL de la base de datos

  Ve a SQL Editor en Supabase y ejecuta los 3 archivos en este orden:

  Primero — supabase/schema.sql
  (crea tablas: products, orders, order_items, storage bucket e imágenes)

  Segundo — supabase/promotions.sql
  (crea tabla: promotions)

  Tercero — supabase/settings.sql
  (crea tabla: store_settings con valores por defecto)

  2.3 Obtener las credenciales

  En Supabase → Settings → API:
  - Copia Project URL → será NEXT_PUBLIC_SUPABASE_URL
  - Copia anon public key → será NEXT_PUBLIC_SUPABASE_ANON_KEY

  - Email: el correo del dueño de la tienda (ej: admin@mitienda.com)

  CLIENTE A (aldea-ofertas)
  ├── Número: +52 7472542065
  ├── Su Phone Number ID en Meta: 1054305881108032
  └── Configura su bot en /admin/whatsapp

  CLIENTE B (otra-tienda)
  ├── Número: +52 XXXXXXXXXX (el suyo)
  ├── Su Phone Number ID en Meta: el que les da Meta
  └── Configura su bot en su /admin/whatsapp

  ---
  Flujo para onboarding de un cliente nuevo

  Cuando contraten tu servicio, los pasos son:

  Tú haces (superadmin):
  1. Crear cliente en /superadmin/clients/new
  2. Ingresar el número de WhatsApp del cliente
  3. Darle acceso al admin (email + contraseña)

  El cliente hace (una sola vez):
  1. Entrar a developers.facebook.com → crear app → agregar WhatsApp
  2. Registrar su número de teléfono
  3. Copiar su Phone Number ID y Access Token
  4. Entrar a su /admin/whatsapp y pegar esos datos
  5. Configurar el webhook con la URL que aparece en esa página

  El webhook URL de cada cliente es el mismo (tu servidor en Railway), pero cada cliente tiene su propio verify_token y phone_number_id, así el sistema sabe a cuál tienda pertenece cada
  mensaje.
