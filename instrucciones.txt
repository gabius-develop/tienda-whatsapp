Paso 1: Crear tu usuario admin en Supabase

  1. Entra a https://supabase.com → tu proyecto                                                                                                
  2. En el menú izquierdo: Authentication → Users                                                                                            
  3. Clic en "Add user" → "Create new user"
  4. Pon el email y contraseña que quieras (por ejemplo admin@mitienda.com y una contraseña segura)
  5. Activa "Auto Confirm User"
  6. Clic en "Create user"

  Esas credenciales son las que usas en /admin/login.

Pasos para poner en producción

  1. Configurar Supabase

  1. Crea proyecto en https://supabase.com
  2. Ve a SQL Editor y pega el contenido de supabase/schema.sql
  3. Ve a Authentication > Users y crea un usuario admin (email + password)

  2. Variables de entorno

  Crea el archivo .env.local (copia .env.local.example):
  NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
  NEXT_PUBLIC_WHATSAPP_PHONE=5212345678901

  3. Probar localmente

  npm run dev
  - Tienda: http://localhost:3000
  - Admin: http://localhost:3000/admin/login

  4. Publicar en Railway

  1. Sube el código a GitHub
  2. En Railway: New Project > Deploy from GitHub
  3. Agrega las variables de entorno en Railway Settings
  4. Deploy automático

  Funcionalidades incluidas

  - Tienda con búsqueda en tiempo real + filtros por categoría
  - Carrito persistido en localStorage (se conserva al cerrar el navegador)
  - Checkout con nombre, teléfono y dirección → abre WhatsApp con pedido formateado
  - Los pedidos se guardan en Supabase (para el ranking)
  - Admin protegido por middleware con Supabase Auth
  - CRUD completo de productos con subida de imágenes a Supabase Storage
  - Dashboard con 4 métricas, top 5 productos más vendidos, últimos pedidos

  push, te volverá a pedir usuario y contraseña/token de la cuenta que necesites.

  Tienda WhatsApp — Guía completa de instalación y despliegue

  ¿Qué es este proyecto?

  Una tienda online donde los clientes pueden ver productos, agregar al carrito y pagar por WhatsApp o MercadoPago. Incluye:

  ┌────────────────┬─────────────┬────────────────────┐
  │     Panel      │     URL     │     Para quién     │
  ├────────────────┼─────────────┼────────────────────┤
  │ Tienda pública │ /           │ Clientes           │
  ├────────────────┼─────────────┼────────────────────┤
  │ Admin          │ /admin      │ Dueño de la tienda │
  ├────────────────┼─────────────┼────────────────────┤
  │ Super Admin    │ /superadmin │ Desarrollador      │
  └────────────────┴─────────────┴────────────────────┘

  Stack: Next.js 16 · TypeScript · Tailwind CSS · Supabase · Zustand · MercadoPago SDK v2

  ---
  PARTE 1 — Clonar el proyecto

  git clone https://github.com/TU_USUARIO/tiendawhatsapp.git
  cd tiendawhatsapp
  npm install

  ---
  PARTE 2 — Configurar Supabase (base de datos)

  2.1 Crear proyecto en Supabase

  1. Ve a https://supabase.com → New project
  2. Elige nombre, contraseña de DB y región (ej: South America - São Paulo)
  3. Espera ~2 minutos a que el proyecto esté listo

  2.2 Ejecutar el SQL de la base de datos

  Ve a SQL Editor en Supabase y ejecuta los 3 archivos en este orden:

  Primero — supabase/schema.sql
  (crea tablas: products, orders, order_items, storage bucket e imágenes)

  Segundo — supabase/promotions.sql
  (crea tabla: promotions)

  Tercero — supabase/settings.sql
  (crea tabla: store_settings con valores por defecto)

  2.3 Obtener las credenciales

  En Supabase → Settings → API:
  - Copia Project URL → será NEXT_PUBLIC_SUPABASE_URL
  - Copia anon public key → será NEXT_PUBLIC_SUPABASE_ANON_KEY

  2.4 Crear usuario admin de la tienda

  En Supabase → Authentication → Users → Add user:
  - Email: el correo del dueño de la tienda (ej: admin@mitienda.com)
  - Password: una contraseña segura
  - Marca "Auto Confirm User"

  Este usuario es quien accede al panel /admin para gestionar productos, pedidos y promociones.

  ---
  PARTE 3 — Variables de entorno locales

  Crea el archivo .env.local en la raíz del proyecto:

  # Supabase
  NEXT_PUBLIC_SUPABASE_URL=https://XXXXXXXX.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...

  # WhatsApp Business (con código de país, sin + ni espacios)
  # México: 521 + 10 dígitos → 5217471234567
  NEXT_PUBLIC_WHATSAPP_PHONE=5217471234567

  # MercadoPago (obtener en mercadopago.com.mx → Credenciales)
  # Para pruebas usa el token TEST-, para producción el token real
  MERCADOPAGO_ACCESS_TOKEN=TEST-XXXXXXXXXXXXXXXXXXXX

  # URL pública de la app (en local usa localhost)
  NEXT_PUBLIC_APP_URL=http://localhost:3000

  # Contraseña del Super Admin (tú como desarrollador)
  SUPERADMIN_PASSWORD=elige-una-contraseña-segura

  Correr localmente

  npm run dev

  Abre http://localhost:3000

  ---
  PARTE 4 — Despliegue en Railway

  4.1 Subir el código a GitHub

  git add .
  git commit -m "initial commit"
  git push origin main

  4.2 Crear proyecto en Railway

  1. Ve a https://railway.app → New Project → Deploy from GitHub repo
  2. Selecciona tu repositorio
  3. Railway detecta automáticamente que es Next.js

  4.3 Agregar variables de entorno en Railway

  Ve a tu proyecto → Variables → agrega todas estas:

  ┌───────────────────────────────┬───────────────────────────────────────────────────────────────┐
  │           Variable            │                             Valor                             │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ NEXT_PUBLIC_SUPABASE_URL      │ URL de tu proyecto Supabase                                   │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ NEXT_PUBLIC_SUPABASE_ANON_KEY │ Anon key de Supabase                                          │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ NEXT_PUBLIC_WHATSAPP_PHONE    │ Número WhatsApp (ej: 5217471234567)                           │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ MERCADOPAGO_ACCESS_TOKEN      │ Token de MercadoPago                                          │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ NEXT_PUBLIC_APP_URL           │ URL que te da Railway (ej: https://tienda-xxx.up.railway.app) │
  ├───────────────────────────────┼───────────────────────────────────────────────────────────────┤
  │ SUPERADMIN_PASSWORD           │ Contraseña que tú elijas para el panel de desarrollador       │
  └───────────────────────────────┴───────────────────────────────────────────────────────────────┘

  ▎ Importante: Cada vez que cambies una variable, Railway hace redeploy automático.

  4.4 Configurar dominio en Railway

  En Railway → tu servicio → Settings → Networking → Generate Domain
  Copia esa URL y ponla en NEXT_PUBLIC_APP_URL.

  ---
  PARTE 5 — Configurar MercadoPago

  Para pruebas (sandbox)

  1. Ve a https://mercadopago.com.mx → inicia sesión
  2. Panel de desarrollador → Credenciales de prueba
  3. Copia el Access Token que empieza con TEST-
  4. Úsalo en MERCADOPAGO_ACCESS_TOKEN

  Para producción (cobros reales)

  1. En MercadoPago → Credenciales de producción
  2. Copia el Access Token real (no empieza con TEST-)
  3. Actualiza MERCADOPAGO_ACCESS_TOKEN en Railway

  ---
  PARTE 6 — Accesos a los paneles

  Panel Admin /admin

  - URL: https://tu-dominio.railway.app/admin/login
  - Email y contraseña: los del usuario creado en Paso 2.4
  - Funciones: agregar/editar productos, ver pedidos, crear promociones, ver estadísticas

  Panel Super Admin /superadmin

  - URL: https://tu-dominio.railway.app/superadmin/login
  - Contraseña: la que pusiste en SUPERADMIN_PASSWORD
  - Funciones: cambiar nombre de la tienda, título de bienvenida, texto del footer

  ---
  PARTE 7 — Flujos de compra

  Cliente ve productos → Agrega al carrito → Llena datos (nombre, teléfono, dirección)
      │
      ├── "Solo por WhatsApp" → Se abre WhatsApp con el pedido formateado
      │
      └── "Pagar con MercadoPago" → Redirige a MP → Paga → Pantalla de éxito
                                                             → Cuenta regresiva 5s
                                                             → Se abre WhatsApp
                                                                con confirmación de pago

  ---
  PARTE 8 — Para un nuevo cliente (checklist)

  Al montar esto para un cliente nuevo, debes:

  - Crear nuevo proyecto en Supabase (pasos 2.1 a 2.4)
  - Ejecutar los 3 archivos SQL
  - Crear el usuario admin con el email del cliente
  - Crear nuevo proyecto en Railway conectado al repo
  - Agregar las 6 variables de entorno
  - Cambiar NEXT_PUBLIC_WHATSAPP_PHONE al número del cliente
  - Agregar el token de MercadoPago del cliente
  - Entrar al Super Admin y personalizar: nombre de tienda, bienvenida, footer
  - Entrar al Admin y cargar los primeros productos con fotos
